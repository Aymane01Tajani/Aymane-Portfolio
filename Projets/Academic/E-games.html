E-Games - Plateforme E-commerce de Jeux Vidéo

Projet Académique Django

Juin 2023 EMSI Casablanca E-commerce

Aperçu du projet

L'Application E-games est une application web permettant aux utilisateurs d'acheter des jeux vidéo en ligne a travers une plateforme e-commerce moderne dédiée à la vente de jeux vidéo développée avec Django. Cette solution complète offre une expérience d'achat fluide avec gestion des utilisateurs, catalogue de produits, panier d'achat, système de paiement et interface d'administration avancée. L'objectif de ce projet est de créer une plateforme conviviale et intuitive offrant différentes fonctionnalités pour faciliter la recherche, la sélection et l'achat des jeux vidéo.

Architecture technique

L'application E-games suit une architecture Modèle-Vue-Template (MVT) pour garantir une séparation claire des responsabilités et favoriser la maintenabilité et l'évolutivité du code. Voici une description de chaque composant de l'architecture :

Modèles (Model)

Les modèles représentent la couche de données de l'application. Dans le cas de l'application E-games, le modèle "Jeu" est utilisé pour stocker les informations sur les jeux disponibles à l'achat, telles que le nom, le prix, la console, etc. Le modèle "Utilisateur" est utilisé pour gérer les informations des utilisateurs enregistrés, telles que le nom, le prénom, l'adresse e-mail, etc. Le modèle "Achat" stocke les détails de chaque achat effectué, comme l'utilisateur, le jeu acheté, le prix, la date, etc.

Vues (View)

Les vues gèrent la logique de l'application et définissent la manière dont les données sont présentées aux utilisateurs. Dans l'application E-games, différentes vues sont développées, telles que la vue de la page d'accueil, la vue de la liste des jeux, la vue des détails d'un jeu, la vue de disponibilité en magasin, la vue du formulaire d'inscription, et la vue de confirmation d'achat. Chaque vue récupère les données nécessaires à partir des modèles, effectue les traitements appropriés et renvoie les résultats aux templates.

Templates (Template)

Les templates sont responsables de l'affichage des données et de la présentation visuelle de l'application. Dans l'application E-games, des templates correspondants sont développés pour chaque vue. Par exemple, il y a un template pour la page d'accueil, un autre pour la liste des jeux, un autre pour les détails d'un jeu, etc. Les templates utilisent des balises et des variables pour afficher les données dynamiques provenant des vues. Ils permettent également d'organiser la structure et le style de l'interface utilisateur.

L'architecture MVT permet de séparer clairement les préoccupations en isolant la logique métier, la présentation et les données de l'application. Cette approche favorise la réutilisabilité du code, facilite les tests et améliore la maintenabilité globale du projet E-games.

Aspects techniques avancés

Base de données

Modélisation relationnelle optimisée avec index sur les recherches fréquentes, relations Many-to-Many pour catégories et tags, système de cache Redis pour améliorer les performances.

Sécurité

Protection CSRF, validation des formulaires, hashage des mots de passe avec bcrypt, sessions sécurisées, et validation des uploads de fichiers.

Performance

Mise en cache des requêtes fréquentes, optimisation des queries ORM, compression des assets statiques, et pagination intelligente des résultats.
