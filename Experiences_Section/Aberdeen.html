Aperçu du projet

Le secteur pharmaceutique est un domaine très réglementé et compétitif qui repose sur la performance des délégués médicaux pour la promotion et la vente des produits. Les méthodes traditionnelles de prise de commandes, souvent manuelles ou obsolètes, engendrent erreurs, lenteurs et manque de visibilité. La transformation digitale s’impose pour optimiser la gestion des commandes et la synchronisation avec les stocks. Dans ce contexte, une application mobile connectée à Sage X3 via un web service REST est développée. Elle permet aux délégués médicaux de gérer leurs clients, consulter les stocks en temps réel, passer des commandes sécurisées et suivre leur avancement.

Architecture technique

Architecture REST distribuée de l'application PharmaX3

L’infrastructure repose sur une architecture distribuée légère, permettant une communication directe et sécurisée entre l’application mobile et Sage X3. Elle est constituée des éléments suivants :

  • Application Mobile (Frontend) : Développée avec Flutter (Dart), cette application est installée sur les tablettes des délégués médicaux. Elle interagit directement avec les Web Services REST exposés par Sage X3 via des requêtes HTTP.
  • Web Services REST Sage X3 : Les services sont configurés à partir de représentations associées à des classes métiers (commandes, clients, stocks). Chaque appel se fait selon les standards suivants : une Authentification HTTP de type Basic Auth à travers un username et un password et un échange de données au format JSON
  • Base de Données (SQL Server) : L’ensemble des données de gestion (clients, articles, commandes, stocks) est centralisé dans la base de données intégrée à Sage X3.

NB : Ce modèle assure un niveau de sécurité basique via les identifiants utilisateurs Sage X3 et garantit l’échange de données structurées en JSON, conforme aux standards REST.
